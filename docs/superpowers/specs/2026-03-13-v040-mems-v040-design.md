# Friedman-cli v0.4.0 Design Spec — Wrapping MEMs v0.4.0

## Overview

Friedman-cli v0.4.0 wraps MacroEconometricModels.jl v0.4.0, adding 6 feature areas:
DSGE historical decomposition, panel regression, spectral analysis,
ordered/multinomial choice models, data enhancements, and dependency updates.

Total: 14 top-level commands (+1: `spectral`), ~199 subcommands (+35 from ~164).

## 2. DSGE Historical Decomposition

### Commands

**`dsge hd`** — Linear DSGE historical decomposition via Kalman smoother.

```
friedman dsge hd <model> --data <csv> --observables <var1,var2,...> \
    [--states=observables|all] [--measurement-error=<csv|auto>] \
    [--plot] [--plot-save=<path>] [-o output] [-f format]
```

Handler: `_dsge_hd(; model, data, observables, states, measurement_error, ...)`

Flow:
1. `_load_dsge_model(model)` → DSGESpec
2. `_solve_dsge(spec)` → DSGESolution
3. `load_data(data)` → DataFrame → `df_to_matrix()` → Matrix
4. Parse `observables` string → `Vector{Symbol}`
5. `historical_decomposition(sol, Y, obs; states, measurement_error)` → HistoricalDecomposition
6. `verify_decomposition(hd)` → print diagnostics
7. Build contributions table per shock → `output_result()`
8. `_maybe_plot()` if --plot/--plot-save

**`dsge bayes hd`** — Bayesian DSGE HD with posterior uncertainty.

```
friedman dsge bayes hd <model> --data <csv> --params <toml> --priors <toml> \
    --observables <var1,var2,...> \
    [--method=smc|rwmh] [--draws=5000] [--mode-only] \
    [--n-draws=200] [--quantiles=0.16,0.5,0.84] \
    [--plot] [--plot-save=<path>] [-o output] [-f format]
```

Handler: `_dsge_bayes_hd(; model, data, params, priors, observables, method, draws, mode_only, n_draws, quantiles, ...)`

Flow:
1. Load model, estimate posterior via `estimate_dsge_bayes()`
2. `historical_decomposition(bd, Y, obs; mode_only, n_draws, quantiles)` → BayesianHistoricalDecomposition
3. Output point_estimate contributions + quantile bounds per shock

### MEMs API Used

```julia
# Linear DSGE HD
historical_decomposition(sol::DSGESolution{T}, data::AbstractMatrix,
    observables::Vector{Symbol}; states=:observables, measurement_error=nothing)
    → HistoricalDecomposition{T}

# Bayesian DSGE HD
historical_decomposition(post::BayesianDSGE{T}, data::AbstractMatrix,
    observables::Vector{Symbol}; mode_only=false, n_draws=200,
    quantiles=[0.16, 0.5, 0.84], measurement_error=nothing)
    → BayesianHistoricalDecomposition{T}

contribution(hd, var, shock)          # accessor
total_shock_contribution(hd, var)     # sum all shocks for variable
verify_decomposition(hd)              # diagnostic
dsge_smoother(sol, data, obs)         # KalmanSmootherResult (internal)
```

### New Types (with full field lists for mock construction)

**`HistoricalDecomposition{T}`** (core/hd.jl — shared with VAR HD, DSGE dispatches into same type):
```julia
struct HistoricalDecomposition{T} <: AbstractHistoricalDecomposition
    contributions::Array{T,3}      # T_eff × n_vars × n_shocks
    initial_conditions::Matrix{T}  # T_eff × n_vars
    actual::Matrix{T}              # T_eff × n_vars
    shocks::Matrix{T}              # T_eff × n_shocks
    T_eff::Int
    variables::Vector{String}
    shock_names::Vector{String}
    method::Symbol
end
```

**`BayesianHistoricalDecomposition{T}`** (core/hd.jl):
```julia
struct BayesianHistoricalDecomposition{T} <: AbstractHistoricalDecomposition
    quantiles::Array{T,4}                   # T_eff × n_vars × n_shocks × n_quantiles
    point_estimate::Array{T,3}              # T_eff × n_vars × n_shocks
    initial_quantiles::Array{T,3}           # T_eff × n_vars × n_quantiles
    initial_point_estimate::Matrix{T}       # T_eff × n_vars
    shocks_point_estimate::Matrix{T}        # T_eff × n_shocks
    actual::Matrix{T}                       # T_eff × n_vars
    T_eff::Int
    variables::Vector{String}
    shock_names::Vector{String}
    quantile_levels::Vector{T}
    method::Symbol
end
```

**`KalmanSmootherResult{T}`** (dsge/smoother.jl):
```julia
struct KalmanSmootherResult{T} <: AbstractAnalysisResult
    smoothed_states::Matrix{T}
    smoothed_covariances::Array{T,3}
    smoothed_shocks::Matrix{T}
    filtered_states::Matrix{T}
    filtered_covariances::Array{T,3}
    predicted_states::Matrix{T}
    predicted_covariances::Array{T,3}
    log_likelihood::T
end
```

Note: `HistoricalDecomposition{T}` and `BayesianHistoricalDecomposition{T}` already exist in mocks
for VAR HD. The DSGE dispatch produces the same types — no new mock types needed for these,
only new mock dispatches of `historical_decomposition()` for `DSGESolution` and `BayesianDSGE` args.
`KalmanSmootherResult` is new.

## 3. Panel Regression

### Estimation Commands (under `estimate`)

**`estimate preg`** — Panel linear regression (FE/RE/FD/Between/CRE/AB/BB).

```
friedman estimate preg <data> --dep <col> \
    [--indep=<col1,col2,...>] \
    [--method=fe|re|fd|between|cre|ab|bb] [--twoway] \
    [--cov-type=ols|cluster|twoway|driscoll-kraay] \
    [--id-col=<col>] [--time-col=<col>] \
    [-o output] [-f format]
```

Handler: `_estimate_preg(; data, dep, indep, method, twoway, cov_type, id_col, time_col, ...)`

Flow:
1. `load_data(data)` → DataFrame
2. `xtset(df, Symbol(id_col), Symbol(time_col))` → PanelData
3. Parse `dep` → `Symbol`, parse `indep` → `Vector{Symbol}` (if empty, use all non-dep numeric cols)
4. `estimate_xtreg(panel, Symbol(dep), indep_syms; model=Symbol(method), twoway, cov_type=Symbol(cov_type))` → PanelRegModel
5. Output coefficient table (beta, SE, t, p, CI) + R2 within/between/overall + F-stat + sigma_u/sigma_e/rho

Note: `--twoway` is a `Flag` (boolean), not an `Option`. `--method` maps to MEMs `model` kwarg.

**Hyphen-to-underscore in option values:** CLI options use hyphens (`driscoll-kraay`, `hausman-taylor`)
but MEMs expects underscore Symbols (`:driscoll_kraay`, `:hausman_taylor`). All handlers must convert:
`model = Symbol(replace(method, "-" => "_"))` and `cov_type_sym = Symbol(replace(cov_type, "-" => "_"))`.

**`estimate piv`** — Panel IV/2SLS.

```
friedman estimate piv <data> --dep <col> --exog <cols> --endog <cols> \
    [--instruments=<cols>] \
    [--method=fe|re|fd|hausman-taylor] \
    [--cov-type=ols|cluster|twoway|driscoll-kraay] \
    [--id-col=<col>] [--time-col=<col>] \
    [-o output] [-f format]
```

Handler: `_estimate_piv(; data, dep, exog, endog, instruments, method, cov_type, id_col, time_col, ...)`

MEMs API: `estimate_xtiv(pd, depvar::Symbol, exog::Vector{Symbol}, endog::Vector{Symbol}; instruments, model, cov_type)`

**`estimate plogit`** — Panel logit.

```
friedman estimate plogit <data> --dep <col> \
    [--indep=<col1,col2,...>] \
    [--method=pooled|fe|re|cre] \
    [--cov-type=ols|cluster] \
    [--id-col=<col>] [--time-col=<col>] \
    [-o output] [-f format]
```

Handler: `_estimate_plogit(; data, dep, indep, method, cov_type, id_col, time_col, ...)`

MEMs API: `estimate_xtlogit(pd, depvar::Symbol, indepvars::Vector{Symbol}; model, cov_type)`

**`estimate pprobit`** — Panel probit.

```
friedman estimate pprobit <data> --dep <col> \
    [--indep=<col1,col2,...>] \
    [--method=pooled|re|cre] \
    [--cov-type=ols|cluster] \
    [--id-col=<col>] [--time-col=<col>] \
    [-o output] [-f format]
```

Handler: `_estimate_pprobit(; data, dep, indep, method, cov_type, id_col, time_col, ...)`

Note: Panel probit does NOT support `--method=fe` (incidental parameters problem).
Only `:pooled`, `:re`, `:cre` are valid. MEMs calls `estimate_xtprobit`.

### Predict / Residuals (under `predict` and `residuals`)

Add leaves: `predict preg`, `predict piv`, `predict plogit`, `predict pprobit`
and `residuals preg`, `residuals piv`, `residuals plogit`, `residuals pprobit`.

All follow: load data → xtset → estimate model → `predict(model)` / `residuals(model)`.

**Note:** `PanelLogitModel` and `PanelProbitModel` do not have a built-in `residuals` field or
StatsAPI `residuals()` dispatch. The handler computes deviance residuals manually:
`r_i = sign(y_i - p_i) * sqrt(-2 * (y_i*log(p_i) + (1-y_i)*log(1-p_i)))`.

### Panel Specification Tests (under `test`)

| Command | MEMs Function | Description |
|---------|--------------|-------------|
| `test hausman` | `hausman_test(fe, re)` | FE vs RE specification |
| `test breusch-pagan` | `breusch_pagan_test(model)` | RE LM test |
| `test f-fe` | `f_test_fe(model)` | F-test for fixed effects |
| `test pesaran-cd` | `pesaran_cd_test(model)` | Cross-sectional dependence |
| `test wooldridge-ar` | `wooldridge_ar_test(model)` | Panel serial correlation |
| `test modified-wald` | `modified_wald_test(model)` | Groupwise heteroskedasticity |

Panel tests require estimating a model first, so each test command accepts
`<data> --dep --indep --id-col --time-col` and estimates internally.

`test hausman` is special: it estimates both FE and RE models, then compares.

## 4. Spectral Analysis (New Top-Level Command)

### Commands

New file: `src/commands/spectral.jl` with `register_spectral_commands!()`.

**`spectral acf`** — Autocorrelation and partial autocorrelation.

```
friedman spectral acf <data> \
    [--column=1] [--max-lag=<int>] [--ccf-with=<col>] \
    [--plot] [--plot-save=<path>] [-o output] [-f format]
```

Handler: `_spectral_acf(; data, column, max_lag, ccf_with, ...)`

Flow:
1. `load_data(data)` → extract column vector
2. `acf(y; maxlag)` → ACFResult (includes ACF, PACF, Q-stats, p-values)
3. If `--ccf-with`, also call `ccf(y, z)`
4. Output table: lag | ACF | PACF | Q-stat | p-value
5. `_maybe_plot()`

**`spectral periodogram`** — Raw periodogram.

```
friedman spectral periodogram <data> [--column=1] \
    [--plot] [--plot-save=<path>] [-o output] [-f format]
```

Handler: `_spectral_periodogram(; data, column, ...)`

**`spectral density`** — Spectral density estimation.

```
friedman spectral density <data> [--column=1] \
    [--method=periodogram|welch|smoothed|ar] [--bandwidth=<float>] \
    [--plot] [--plot-save=<path>] [-o output] [-f format]
```

Handler: `_spectral_density(; data, column, method, bandwidth, ...)`

**`spectral cross`** — Cross-spectral analysis.

```
friedman spectral cross <data> --var1=<col> --var2=<col> \
    [--plot] [--plot-save=<path>] [-o output] [-f format]
```

Handler: `_spectral_cross(; data, var1, var2, ...)`

Output: freq | co-spectrum | quad-spectrum | coherence | phase | gain

**`spectral transfer`** — Filter transfer function.

```
friedman spectral transfer [--filter=hp|bk|hamilton|ideal] \
    [--lambda=<float>] [--nobs=<int>] \
    [--plot] [--plot-save=<path>] [-o output] [-f format]
```

Handler: `_spectral_transfer(; filter, lambda, nobs, ...)`

Note: No `<data>` positional arg — computes theoretical transfer function.

## 5. Ordered & Multinomial Choice Models

### Estimation Commands (under `estimate`)

**`estimate ologit`** — Ordered logistic regression.

```
friedman estimate ologit <data> --dep <col> \
    [--cov-type=hc0|hc1|hc2|hc3|cluster] [--clusters=<col>] \
    [-o output] [-f format]
```

Handler: `_estimate_ologit(; data, dep, cov_type, clusters, ...)`

Flow:
1. `_load_reg_data(data, dep)` → (y, X, varnames)
2. `estimate_ologit(y, X; cov_type, varnames)` → OrderedLogitModel
3. Output: cutpoints + slope coefficients table (beta, SE, z, p, CI) + pseudo-R2 + AIC/BIC

**`estimate oprobit`** — Ordered probit. Same interface, calls `estimate_oprobit`.

**`estimate mlogit`** — Multinomial logit.

```
friedman estimate mlogit <data> --dep <col> \
    [--cov-type=hc0|hc1|hc2|hc3] \
    [-o output] [-f format]
```

Handler: `_estimate_mlogit(; data, dep, cov_type, ...)`

Output: Per-category coefficient table + pseudo-R2 + AIC/BIC.

Note: MEMs `estimate_mlogit(y, X; ...)` uses the first sorted unique value as base category.
No `--base-category` option — the base category is determined automatically.

### Predict / Residuals

- `predict ologit` / `predict oprobit` — predicted probabilities per category
- `predict mlogit` — predicted probabilities per category
- `residuals ologit` / `residuals oprobit` / `residuals mlogit` — deviance residuals

### Tests (under `test`)

| Command | MEMs Function | Description |
|---------|--------------|-------------|
| `test brant` | `brant_test(model)` | Parallel regression (ordered models) |
| `test hausman-iia` | `hausman_iia(model; omit_category)` | IIA for multinomial logit |

`test brant` accepts `<data> --dep` and estimates an ordered logit internally before testing.

`test hausman-iia` accepts `<data> --dep --omit-category=<int>` and estimates multinomial logit
internally before testing. The `--omit-category` option specifies which category to omit
for the Hausman IIA test (required).

### MEMs API Used

```julia
estimate_ologit(y, X; cov_type, varnames) → OrderedLogitModel
estimate_oprobit(y, X; cov_type, varnames) → OrderedProbitModel
estimate_mlogit(y, X; cov_type, varnames) → MultinomialLogitModel
brant_test(model::OrderedLogitModel) → test result
hausman_iia(model::MultinomialLogitModel; omit_category::Int) → test result
marginal_effects(model)  # for ordered/multinomial
```

## 6. Data Enhancements

### New Leaves (under `data`)

**`data dropna`** — Drop rows with missing/NaN values.

```
friedman data dropna <data> [--vars=<col1,col2,...>] [-o output] [-f format]
```

Handler: `_data_dropna(; data, vars, ...)`

Flow:
1. `load_data(data)` → DataFrame → `TimeSeriesData`
2. Parse `--vars` to `Vector{String}` or `nothing`
3. `dropna(ts; vars)` → cleaned TimeSeriesData
4. Output cleaned data summary (rows before/after) + data table

**`data keeprows`** — Filter rows by index range.

```
friedman data keeprows <data> --rows=<range> [-o output] [-f format]
```

Handler: `_data_keeprows(; data, rows, ...)`

Flow:
1. `load_data(data)` → DataFrame → `TimeSeriesData`
2. Parse `--rows` string to `Vector{Int}` (supports "1:100", "1,5,10", "50:end")
3. `keeprows(ts, indices)` → filtered TimeSeriesData
4. Output filtered data

Note: `keeprows` in MEMs accepts `Vector{Int}` or `BitVector`. The CLI parses a row
range string to `Vector{Int}` — no expression evaluation needed.

### MEMs API Used

```julia
dropna(ts::TimeSeriesData; vars=nothing) → TimeSeriesData
dropna(pd::PanelData; vars=nothing) → PanelData
dropna(cs::CrossSectionData; vars=nothing) → CrossSectionData
keeprows(ts::TimeSeriesData, idx::Vector{Int}) → TimeSeriesData
keeprows(pd::PanelData, idx::Vector{Int}) → PanelData
keeprows(cs::CrossSectionData, idx::Vector{Int}) → CrossSectionData
```
